# Week 1 Execution Checklist
## Project Sitel — Zero to First Paying Client in 7 Days

**Mindset:** Speed beats perfection this week. A working demo that's 80% polished sent to 20 prospects is infinitely more valuable than a perfect demo sent to nobody. Ship, iterate, close.

---

## DAY 1 — MONDAY: Build the Foundation

**Theme: Get the demo live. Nothing else matters until this is done.**

### Morning (9am–1pm) — Retell AI Setup

- [ ] Sign up for Retell AI at app.retellai.com (30 min)
- [ ] Create new LLM agent — name it "Sitel Demo — eCommerce Support" (10 min)
- [ ] Paste system prompt from `demo-script.md` into agent (15 min)
- [ ] Set model to GPT-4o, set begin message, configure end-call phrases (10 min)
- [ ] Select Charlotte voice (ElevenLabs) — test playback (10 min)
- [ ] Sign up for Twilio at twilio.com (15 min)
- [ ] Get UK phone number (London 020 preferred) (10 min)
- [ ] Connect Twilio number to Retell AI (20 min)
- [ ] Run first internal test via Retell's built-in test tool (15 min)
- [ ] Make first real call to the number from your mobile (15 min)

**Milestone by 1pm: You have a working phone number you can call.**

### Afternoon (2pm–6pm) — Polish and LinkedIn

- [ ] Test all 5 scenarios from `demo-script.md`:
  - [ ] Order tracking (#UK100234)
  - [ ] Returns flow
  - [ ] FAQ (delivery times)
  - [ ] Complaint/de-escalation
  - [ ] Human escalation trigger
- [ ] Fix any issues in the system prompt (iterate 2–3 times if needed)
- [ ] Record a call and listen back — does it sound natural?
- [ ] **LinkedIn profile audit:**
  - [ ] Profile photo: professional, clear face, good lighting
  - [ ] Headline: "Founder @ Sitel AI | We deploy AI voice agents for eCommerce brands | Replacing Tier-1 call centres"
  - [ ] About section: 3 paragraphs — problem you solve, how you solve it, who you work with. No jargon.
  - [ ] Experience: add Sitel AI as current role with description
  - [ ] Banner image: if possible, something with "AI Voice Support" or your brand
- [ ] Set up a simple spreadsheet to track prospects (copy template from `icp-targeting.md`)

**Day 1 done when:** Demo number works. LinkedIn looks like a real business. You're ready to reach out tomorrow.

---

## DAY 2 — TUESDAY: First 20 Outreach Messages

**Theme: Fill the pipeline. Volume matters today.**

### Morning (9am–12pm) — Prospect Research

- [ ] Run LinkedIn Sales Navigator search (filters from `icp-targeting.md`)
- [ ] Identify and score 40 target prospects (use scoring framework in `icp-targeting.md`)
- [ ] Check each brand's Trustpilot page — screenshot any complaint themes
- [ ] Check each brand's LinkedIn Jobs page — note if hiring CS agents
- [ ] Build your priority list: top 20 scored prospects to contact today

### Afternoon (12pm–6pm) — Send Outreach

- [ ] Send 20 personalised LinkedIn messages (use templates from `outreach-messages.md`)
  - Personalise the first sentence for each one
  - Reference a specific pain signal (Trustpilot / hiring / growth)
  - Include the demo phone number
  - Aim to send one every 15 minutes — don't rush them but don't overthink
- [ ] Send 5 cold emails to targets you found email addresses for
- [ ] Set calendar reminder for Day 5 (follow-up Day 3 messages)
- [ ] Log every outreach in your spreadsheet with date, platform, message variant used

**Day 2 done when:** 20+ personalised messages sent. Pipeline is live.

**Mindset note:** The demo number is doing your selling for you. Your job is to get people to call it. Don't try to close on the first message.

---

## DAY 3 — WEDNESDAY: Follow-Ups + Refine the Demo

**Theme: The people who called the demo yesterday are your hottest leads.**

### Morning (9am–11am) — Check Activity

- [ ] Check Twilio call logs — did anyone call the demo number?
  - Log who called (number), when, how long they stayed on
  - Anyone who stayed 3+ minutes: LinkedIn message them immediately
- [ ] Check LinkedIn — any replies, profile views, message reads?
  - Reply to every response within 2 hours (same day)
  - For positive replies: book a call this week (Wednesday PM or Thursday)
  - For "not right now" replies: polite acknowledgement, park for 90 days
  - For questions: answer briefly and push to a call
- [ ] Check email inbox for cold email replies

### Morning (11am–1pm) — Listen to Demo Calls

- [ ] Pull any recorded calls from Retell AI dashboard
- [ ] Listen to every call in full
- [ ] Note: any awkward pauses? Any wrong answers? Any unnatural phrasing?
- [ ] Update the system prompt to fix any issues
- [ ] Test again after changes

### Afternoon (2pm–5pm) — More Outreach + Prep

- [ ] Send 10 more LinkedIn messages to prospects from your researched list
- [ ] Send Day 3 follow-ups to anyone from Day 1 who hasn't replied (use `outreach-messages.md` Day 3 template)
- [ ] Prepare for any discovery calls booked:
  - Review the company on LinkedIn and Trustpilot
  - Note their estimated call volume and team size
  - Have call notes template open (`discovery-call-script.md`)
  - Have demo number ready
  - Have pricing sheet open (`pricing.md`)

**Day 3 done when:** Responded to all replies. Demo refined. At least 1 discovery call booked.

---

## DAY 4 — THURSDAY: Discovery Calls

**Theme: Run the best possible discovery calls. Get to next steps.**

### For Each Discovery Call

- [ ] 5 minutes before: review the company, open `discovery-call-script.md`
- [ ] Run the call — follow the script structure, not word for word
- [ ] During the demo section: give them the number, stay quiet while they call
- [ ] Book next step on the call — proposal review, not "I'll send it over"
- [ ] Fill in call notes template immediately after hanging up
- [ ] Send LinkedIn/email follow-up within 2 hours:

> "Really enjoyed the call today — I'll have your proposal over by [DATE]. In the meantime, here's the demo number if you want to share it with your team: [NUMBER]. Speak soon."

### If No Discovery Calls Booked Yet

- [ ] Send 20 more outreach messages
- [ ] Try a different signal — shift from hiring signal to Trustpilot signal or vice versa
- [ ] Try a different tier of company — go up or down in company size
- [ ] DM 5 prospects you've already messaged on a different platform (email if you LinkedIn messaged them)

**Day 4 done when:** At least 1 discovery call completed OR 20 more messages sent. Proposals to write tomorrow.

---

## DAY 5 — FRIDAY: Proposals Out

**Theme: Any discovery call from this week needs a proposal today.**

### Morning — Write Proposals

- [ ] For each qualified prospect from discovery calls:
  - [ ] Open `proposal-template.md`
  - [ ] Fill in their specific numbers (call volume, agents, cost)
  - [ ] Calculate their ROI using the formula in `pricing.md`
  - [ ] Choose the right tier based on their call volume
  - [ ] Export as PDF (clean, professional)
  - [ ] Double-check all figures — no errors in a proposal
- [ ] Send proposal with this email:

**Subject:** Your Sitel AI Proposal — [COMPANY NAME]

> Hi [NAME],
>
> As promised — attached is the proposal for [COMPANY]. I've based the numbers on what you shared on Thursday: [1 line summary of their situation].
>
> The short version: [Starter/Growth/Scale] plan at £[MONTHLY]/month, with a payback on the setup fee in [X] weeks based on current staffing costs.
>
> I've blocked time [DATE/TIME] for us to walk through it together — let me know if that still works or if you'd prefer a different slot.
>
> [YOUR NAME]

### Afternoon — Keep the Funnel Full

- [ ] Send Day 7 follow-ups to Day 1 non-responders (final chase)
- [ ] Send 10 more outreach messages to new prospects
- [ ] Review your pipeline spreadsheet: where is everyone?
  - Outreach sent: how many
  - Replies received: how many
  - Demo calls completed: how many
  - Discovery calls completed: how many
  - Proposals sent: how many
- [ ] Identify your 3 hottest prospects — what's the next action for each?

**Day 5 done when:** At least 1 proposal sent. Pipeline reviewed.

## DAY 6–7 — WEEKEND: Close Attempt

**Theme: Stay in motion. First client could come from anywhere.**

### Saturday

- [ ] Check LinkedIn and email — reply to anything over the weekend (fast replies build trust)
- [ ] If anyone has received a proposal: WhatsApp or email "Hope you've had a chance to look at the proposal — happy to jump on a quick call over the weekend or Monday if you have questions"
- [ ] Research next week's outreach list — identify 50 more prospects to approach next week
- [ ] Review what worked this week:
  - Which message variation got the most replies?
  - Which industries responded fastest?
  - What objections came up?
  - What would you change in the demo?

### Sunday

- [ ] Prepare for next week:
  - [ ] Week 2 outreach list built and prioritised
  - [ ] Proposal template reviewed — any improvements based on feedback?
  - [ ] Discovery call script updated based on how calls went
  - [ ] Demo script refined if any edge cases came up
- [ ] Write one paragraph reflecting on the week: what moved the needle, what didn't

## WEEK 1 TARGET METRICS

| Metric | Target |
|---|---|
| Demo live and working | Day 1 ✓ |
| Outreach messages sent | 50+ |
| LinkedIn replies received | 3–5 |
| Demo number calls from prospects | 5–10 |
| Discovery calls completed | 2–3 |
| Proposals sent | 1–2 |
| Signed contracts | 0–1 (if 1: you've won) |

**Don't be disheartened if no contract is signed in Week 1.** A typical B2B sales cycle for a £5K–£15K product is 2–4 weeks from first contact to signature. If you've built a pipeline of 3+ active conversations by the end of Week 1, you're on track.

---

## IF YOU'RE STUCK

**No replies to outreach:** Change the opening line. Try a more specific pain signal. Make the message 20% shorter.

**Replies but no calls booked:** The message is working but the CTA isn't. Add urgency: "I have a slot Thursday at 2pm — want me to send a calendar invite?"

**Calls booked but no proposals:** The demo didn't land or the qualification failed. Listen to the call recording, review `discovery-call-script.md`, adjust.

**Proposals sent but no close:** Follow up every 3 days. Add social proof as it develops. Address the specific objection they haven't voiced yet.

**The demo agent sounds bad:** Go back to `retell-setup-guide.md`, tweak the prompt, test again. A bad demo kills everything downstream.
